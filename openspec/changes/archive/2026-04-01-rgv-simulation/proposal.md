## Why

工廠倉儲人員需要在規劃 RGV（Rail Guided Vehicle）軌道與庫位佈局時，能夠預估每次取放料作業的時間，以評估產能與設備效率。目前缺乏可視化、可互動的模擬工具，導致規劃時只能靠人工估算，誤差大且耗時。

## What Changes

- 全新建立 React + TypeScript + SVG 的 RGV 模擬 Web 應用程式
- 提供 RGV 設定介面（尺寸、初始位置、三軸運動參數、動作模式）
- 提供軌道設定介面（長度、寬度）
- 提供庫位管理介面（雙側配置、1/2 層、每層取放料高度與深度）
- 實作含加減速物理模型的精確時間計算引擎
- 提供序列與疊加兩種動作模式的時間計算
- 提供 SVG 動畫視圖（俯視圖 + 側視圖）
- 提供作業歷史紀錄功能

## Capabilities

### New Capabilities

- `rgv-config`: RGV 參數設定 — 尺寸、初始位置、三軸（行走/升降/進退）各自的最高速、加速度、減速度，以及動作模式（序列/疊加）
- `track-config`: 軌道設定 — 長度、寬度（mm）
- `storage-management`: 庫位管理 — 新增/編輯/刪除庫位，設定 X 位置、側邊（左/右）、層數（1/2層）、每層的取料高度/深度與放料高度/深度
- `time-calculation`: 時間計算引擎 — 使用梯形/三角速度曲線的加減速物理模型，計算每步驟時間並輸出明細；支援序列模式（步驟加總）與疊加模式（關鍵路徑）
- `simulation-view`: SVG 動畫視圖 — 俯視圖（RGV 沿軌道移動）與側視圖（牙叉升降/進退），含播放/暫停/重置及速度倍率控制
- `task-execution`: 任務執行面板 — 選擇取料庫位+層、放料庫位+層，執行模擬並顯示時間明細
- `history-log`: 作業歷史紀錄 — 記錄每次模擬的來源、目標、總時間、時間戳

## Impact

- 全新專案，無既有程式碼影響
- 依賴：React 18、TypeScript、Tailwind CSS、Vite
- 純前端應用，無後端/資料庫需求
- 執行環境：現代瀏覽器（Chrome / Edge / Firefox）
